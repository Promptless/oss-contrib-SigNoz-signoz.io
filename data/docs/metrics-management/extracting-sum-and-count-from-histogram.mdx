A histogram metric like `myApp_latency_bucket` contains detailed bucket data (for example, how many requests fell into each latency range). Often, you only need two numbers from it:
- **Count** — the total number of observations
- **Sum** — the total combined value of all observations

With those two values you can calculate the **average** (`sum ÷ count`), and you no longer need the full bucket breakdown.
This guide shows how to:
1. Extract the count and sum from a histogram into their own metrics.
2. Drop the original histogram so you're not storing unnecessary bucket data.
3. Use the resulting metrics for average calculations

## Configuration
This requires **two separate processors** in your OpenTelemetry Collector pipeline:
- The **transform** processor to extract count and sum.
- The **filter** processor to drop the original histogram.

```yaml
processors:
    transform:
      error_mode: ignore
      metric_statements:
        - context:
            metric:
              conditions:
                - name == "myApp_latency_bucket"
              statements:
                - extract_count_metric(true)
                - extract_sum_metric(true)
    filter:
      metrics:
        exclude:
          match_type: strict
          metric_names:
            - myApp_latency_bucket

service:
  pipelines:
    metrics:
      processors: [transform, filter]
```

## Generated Metrics

The transformation will create two new metrics:

1. myApp_latency_bucket_count: Total number of observations
2. myApp_latency_bucket_sum: Sum of all observations

You can then compute the average latency:
average latency = myApp_latency_bucket_sum ÷ myApp_latency_bucket_count
