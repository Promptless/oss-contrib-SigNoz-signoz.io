If you use Logstash to collect logs in your stack, you can forward them to SigNoz via the OpenTelemetry Collector using the TCP protocol.

<KeyPointCallout title="Using self-hosted SigNoz?" defaultCollapsed="true">
  Most steps are identical. To adapt this guide, update the endpoint and remove the ingestion key header as shown in [Cloud → Self-Hosted](https://signoz.io/docs/ingestion/cloud-vs-self-hosted/#cloud-to-self-hosted).
</KeyPointCallout>

## Prerequisites

- A running Logstash instance
- An instance of SigNoz (either [Cloud](https://signoz.io/teams/) or [Self-Hosted](https://signoz.io/docs/install/self-host/))

## Send Logs to SigNoz

<TabItem value="vm" label="VM" default>

<KeyPointCallout title="What classifies as VM?" defaultCollapsed="true">
A VM is a virtual computer that runs on physical hardware. This includes:
- **Cloud VMs**: AWS EC2, Google Compute Engine, Azure VMs, DigitalOcean Droplets
- **On-premise VMs**: VMware, VirtualBox, Hyper-V, KVM
- **Bare metal servers**: Physical servers running Linux/Unix directly

Use this section if you're deploying your application directly on a server or VM without containerization.

</KeyPointCallout>

### Step 1: Add OpenTelemetry Collector Binary

Add the OpenTelemetry Collector binary to your VM by following the [OTel binary setup guide](https://signoz.io/docs/tutorial/opentelemetry-binary-usage-in-virtual-machine/).

### Step 2: Configure the tcplog Receiver

Add the following to your `config.yaml`:

```yaml
receivers:
  tcplog/logstash:
    max_log_size: 1MiB
    listen_address: '0.0.0.0:2256'

exporters:
  otlp:
    endpoint: "https://ingest.<region>.signoz.cloud:443"
    headers:
      signoz-ingestion-key: "<your-ingestion-key>"

service:
  pipelines:
    logs:
      receivers: [otlp, tcplog/logstash]
      processors: [batch]
      exporters: [otlp]
```

Verify these values:

- `<region>`: Your [SigNoz Cloud region](https://signoz.io/docs/ingestion/signoz-cloud/overview/#endpoint).
- `<your-ingestion-key>`: Your SigNoz [ingestion key](https://signoz.io/docs/ingestion/signoz-cloud/keys/).
- Port `2256` is used here, but you can use any available port.
- Learn more about the <a href="https://github.com/open-telemetry/opentelemetry-collector-contrib/tree/main/receiver/tcplogreceiver" target="_blank" rel="noopener noreferrer nofollow">tcplog receiver</a>.

### Step 3: Update Logstash Configuration

Add the following `output` block to your existing Logstash configuration to forward logs to the OpenTelemetry Collector:

```ruby
output {
  tcp {
    codec => json_lines # Ensures logs are sent in JSON format line-by-line
    host => "localhost"
    port => 2256
  }
}
```

- This config assumes Logstash is running on the same host as the Collector. Adjust the `host` value if Logstash is running elsewhere.

### Step 4: Start the Services

Restart the OpenTelemetry Collector and Logstash.

### Step 1: Create the OTel Collector Config

Create a file named `otel-collector-config.yaml` with the `tcplog` receiver:

```yaml:otel-collector-config.yaml
receivers:
  tcplog/logstash:
    max_log_size: 1MiB
    listen_address: '0.0.0.0:2256'

processors:
  batch:

exporters:
  otlp:
    endpoint: "https://ingest.<region>.signoz.cloud:443"
    headers:
      signoz-ingestion-key: "<your-ingestion-key>"

service:
  pipelines:
    logs:
      receivers: [tcplog/logstash]
      processors: [batch]
      exporters: [otlp]
```

Verify these values:

- `<region>`: Your [SigNoz Cloud region](https://signoz.io/docs/ingestion/signoz-cloud/overview/#endpoint).
- `<your-ingestion-key>`: Your SigNoz [ingestion key](https://signoz.io/docs/ingestion/signoz-cloud/keys/).
- Learn more about the <a href="https://github.com/open-telemetry/opentelemetry-collector-contrib/tree/main/receiver/tcplogreceiver" target="_blank" rel="noopener noreferrer nofollow">tcplog receiver</a>.

### Step 2: Create the Logstash Config

Create a `logstash.conf` file with an `input` to receive logs and an `output` to forward them to the OTel Collector:

```ruby:logstash.conf
input {
  file {
    path => "/var/log/app/*.log"
    start_position => "beginning"
  }
}

output {
  tcp {
    codec => json_lines
    host => "otel-collector"
    port => 2256
  }
}
```

- Replace the `input` block with the source that matches your setup. The `file` input above is just an example.
- `otel-collector` is the Docker service name defined in the next step. Docker networking resolves this to the correct container automatically.

<KeyPointCallout title="Other common Logstash inputs" defaultCollapsed="true">
Replace the `file` input above with whichever source suits your environment:

```ruby
# Receive logs from Filebeat agents
input { beats { port => 5044 } }

# Receive logs over TCP from an application
input { tcp { port => 5000 codec => json_lines } }

# Receive syslog messages
input { syslog { port => 514 } }

# Read from stdin (useful for quick testing)
input { stdin { } }
```

See the <a href="https://www.elastic.co/guide/en/logstash/current/input-plugins.html" target="_blank" rel="noopener noreferrer nofollow">Logstash input plugins</a> documentation for the full list.
</KeyPointCallout>

### Step 3: Create/Update the Docker Compose file

```yaml:docker-compose.yaml
services:
  otel-collector:
    image: otel/opentelemetry-collector-contrib:latest
    volumes:
      - ./otel-collector-config.yaml:/etc/otelcol-contrib/config.yaml
    ports:
      - "2256:2256"

  logstash:
    ...
    volumes:
      - ./logstash.conf:/usr/share/logstash/pipeline/logstash.conf
      - /var/log/app:/var/log/app:ro # Mount the log directory referenced in logstash.conf
    depends_on:
      - otel-collector
```

- Adjust the volume mounts to match the log paths used in your `logstash.conf` input configuration.

### Step 4: Start the Services

```bash
docker compose up -d
```

## Validate

Once the services are running:

1. Open SigNoz and navigate to **Logs** > **Logs Explorer**.
2. Search for logs by filtering on your service or source. Logstash logs should start appearing within a few seconds.
3. Click on a log entry and verify it contains the expected fields from your Logstash pipeline (e.g., `message`, `host`, any custom fields).

## Troubleshooting

</ToggleHeading>

#### Logs are not appearing in SigNoz

- Verify that the OpenTelemetry Collector is running and the `tcplog/logstash` receiver is enabled in both the `receivers` section and the `service.pipelines.logs` pipeline.
- Confirm that Logstash is sending logs to the correct host and port (e.g., `localhost:2256`).
- Check the OpenTelemetry Collector logs for errors related to the `tcplog` receiver.
- After fixing, restart both services and check **Logs** > **Logs Explorer** in SigNoz to confirm logs appear.

#### Port conflict

- If port `2256` is already in use, choose a different port. Update the port in both the Collector `config.yaml` and the Logstash output configuration.
- After updating, restart both services and verify logs appear in **Logs** > **Logs Explorer**.

#### Logs appear but are not parsed correctly

- Ensure the Logstash `output` block uses `codec => json_lines` so logs are sent in JSON format line-by-line.
- Use [Log Pipelines](https://signoz.io/docs/logs-pipelines/introduction/) in SigNoz to parse and transform log fields.
- After updating, check **Logs** > **Logs Explorer** and confirm the log fields are parsed as expected.
